# 宵（Yoi）サポートページ 設計書

最終更新: 2026-06-28

## 背景・目的

iPhone アプリ「宵（Yoi）」（リポ／コードネーム: `restward`）をリリースするにあたり、
App Store Connect の「サポート URL」に登録できるサポートページが必要になった。
独自ドメインを取得するほどではないため、既存サイト `nyaion.tech` 配下に静的サブページとして設置する。

宵（Yoi）は持久系アスリート向けの休養支援アプリ。HealthKit の記録から休養の目安を見守り、
競技者に「休む許可」を与える番人がコンセプト。ローカル完結・アカウント不要・iPhone 専用・MVP 段階。
健康に隣接するため、**医療診断・医療指示ではない**旨の免責をページの要点として明示する。

## スコープ

### やること
- `apps/yoi/index.html` を新規追加し、`https://nyaion.tech/apps/yoi/` でサポートページを公開する。
- 既存 `legal/` サブページと同じ静的 HTML + 共有 `styles.css` 方式に揃える。
- `legal/privacy/` の「アプリ別の取り扱い」表に宵（Yoi）の行を追加する。

### やらないこと（YAGNI）
- 新規 CSS ファイル・新規スタイルクラスの追加（既存 `legal-*` クラスを流用）。
- 料金に関する記載（標準構成では含めない。必要になれば別途追加）。
- 対応 iOS バージョンの数値直書き（申請前後で変動するため「App Store ページ参照」とする）。
- ポータル `index.html`（トップ）からのリンク追加（今回の必須要件ではない）。
- 多言語化・お問い合わせフォーム等の動的機能。

## 配置とルーティング

- 追加ファイル: `apps/yoi/index.html`（2 階層深さ。`legal/privacy/` と同じ相対参照になる）。
- 公開 URL: `https://nyaion.tech/apps/yoi/`。
- 配信: 既存の Cloudflare Pages 設定（`wrangler.toml` の `pages_build_output_dir = "."`）のまま。追加設定不要。
- キャッシュ: `_headers` の `/*` ルール（`Cache-Control: no-cache`）が全パスに適用されるため追記不要。
- App Store Connect の「サポート URL」に上記 URL を登録する（運用作業。コード変更外）。

## スタイル方針

- 既存 `styles.css` の `legal-*` レイアウトクラス（`legal-page` / `legal-header` / `legal-nav` /
  `legal-hero` / `legal-kicker` / `legal-section` / `legal-table` / `legal-footer` 等）を**そのまま流用**する。
- スタイル参照は legal サブページと同じく `<link rel="stylesheet" href="../../styles.css?v=2026062201">`。
  styles.css は変更しないため既存のバージョンクエリ値を踏襲する。
- `legal-*` というクラス名は法務ページ由来だが、レイアウトは汎用的なドキュメント体裁であり流用に問題はない。
  将来セマンティクスを厳密化したくなった場合のみクラスの一般化を検討する（今回は対象外）。

## ページ構成（標準）

`<body class="legal-page">` 内に以下を配置する。文言はすべて日本語。

1. **ヘッダー**（`legal-header` + `legal-brand`）
   - `nyaiontech` ワードマーク → ポータル（`../../`）へのリンク。
   - ナビは最小限（必要なら「サポート」現在地ラベルのみ）。

2. **ヒーロー**（`legal-hero`）
   - キッカー: `Support`。
   - 見出し: 「宵 — Yoi」。
   - リード文: 持久系アスリート向けの休養支援アプリ。HealthKit の記録から休養の目安を見守り、「休む許可」を与える番人であること。
   - 最終更新日（`legal-updated`）。

3. **このアプリについて**（`legal-section`）
   - 番人コンセプトの紹介を 2〜3 文で。「休養はサボりではない。休養はトレーニングの一部である。」の趣旨を平易に。

4. **よくある質問（FAQ）**（`legal-section` の連なり、各設問を小見出し or 定義リストで）
   - HealthKit の許可はなぜ必要か／許可しないとどう動くか。
   - データはどこに保存されるか（**ローカル完結・サーバー送信なし・アカウント不要**）。
   - 休養の目安はどう決まるか（HealthKit の記録に基づく**目安であり、医療診断・体調保証ではない**）。
   - データを削除したいとき（アプリ削除でアプリ内データは消える／HealthKit 連携は「設定」アプリから解除できる旨）。
   - 対応端末（iPhone。対応 iOS バージョンは App Store ページを参照）。

5. **医療・健康に関する免責**（`legal-section`、独立セクションで強調）
   - 本アプリは情報提供を目的とし、医療診断・治療・医療指示を目的としない。
   - 体調に不安がある場合は医療専門家に相談する旨。

6. **問い合わせ**（`legal-section`）
   - 連絡先: `contact@nyaion.tech`（`mailto:`）。
   - 返信は数日かかる場合がある旨を一言。

7. **フッター**（`legal-footer`）
   - Privacy（`../../legal/privacy/`）／ Terms（`../../legal/terms/`）／ nyaiontech ポータル（`../../`）へのリンク。

### `<head>` メタ
- `lang="ja"`、`charset`、`viewport`。
- `<title>`: `サポート | 宵 Yoi | Nyaiontech`（既存 legal ページの命名に倣う）。
- `<meta name="description">`: 宵（Yoi）のサポート情報である旨。
- `<link rel="canonical" href="https://nyaion.tech/apps/yoi/">`。

## 受け入れ条件（Acceptance Criteria）

- `apps/yoi/index.html` が存在し、`legal-page` レイアウトで崩れなく表示される。
- ローカル確認（`just` / プレビュー）で `/apps/yoi/` が 200 で開け、CSS が適用される。
- ページ内の全リンク（ポータル・Privacy・Terms・mailto）が正しい相対先を指す。
- 「ローカル完結／アカウント不要」「医療診断ではない」旨が本文に明記されている。
- `legal/privacy/` の表に宵（Yoi）の行が追加され、既存の Beaconlog 行と整合した記述になっている。
- 推測値（料金・iOS バージョン）を直書きしていない。
